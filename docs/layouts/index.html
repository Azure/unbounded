How It Works

Four provisioning paths, one control plane.

machina

Provisions remote machines over SSH. Point it at a Linux host and it handles kubelet install, bootstrap, and node lifecycle.

SSH Guide →

cloud‑init

Spin up GPU and compute instances on Nebius, CoreWeave, OCI, Azure, AWS, and more. Nodes join automatically on first boot via cloud‑init user‑data.

Cloud API Guide →

metalman

PXE-boots bare-metal servers with bundled DHCP, TFTP, and HTTP. Integrates Redfish BMC and TPM 2.0 attestation.

Bare Metal →

unbounded-net

CNI plugin providing pod-to-pod connectivity across sites via WireGuard, GENEVE, VXLAN, or IPIP tunnels.

Networking →

Get Up and Running

  1. 1
    Install the kubectl plugin $ kubectl unbounded --help
  2. 2
    Initialize a site — sets up networking and the machina controller $ kubectl unbounded site init --name mysite ...
  3. 3
    Add machines — via SSH, cloud-init, or PXE $ kubectl unbounded site add-machine --host 10.0.0.5 ...
  4. 4
    Watch them join — Pending → Provisioning → Joining → Ready $ kubectl get machines -w
